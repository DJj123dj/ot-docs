## ‚öô Configuration \{#config}
Open Ticket is fully configurable using <C color="blue">**five JSON config files**</C>.
Each file controls a different aspect of the bot, allowing you to customize everything from ticket options to panel layouts.
Here‚Äôs a breakdown of each config file and what they do:  

### üíº [`config/general.json`](/docs/reference/config/general.mdx) ‚Äì General Bot Settings \{#config-general}
This file contains the **core settings** of Open Ticket, including:  
- **Bot token** - never share this 
- **Bot Status** - configure the bot status 
- **Primary Color** - for embeds and messages  
- **Permissions** - who can manage tickets
- **Extra settings** - button visibility, emojis & additional settings

üìå *This is the most important config file!*  

### ‚ùì [`config/questions.json`](/docs/reference/config/questions.mdx) ‚Äì Custom Ticket Questions \{#config-questions}
This file defines **questions** that users must answer before opening a ticket.  
- Each question is shown in a form (modal) before ticket creation.  
- You can add **up to 5 questions** per ticket type.  
- Questions can have custom names, placeholders, and input limits.  

üìå *If no questions are configured, tickets will open immediately!*  

### üéü [`config/options.json`](/docs/reference/config/options.mdx) ‚Äì Ticket Types & Buttons \{#config-options}
This file defines the **ticket options**, which appear as buttons or dropdown items in your panel.  
- Each option represents a **ticket type, website link, or reaction role**.  
- You can customize **admins, embeds, and button layouts** per option.  
- Options can be linked to **specific questions** (from `questions.json`).  
- Options can be used in **panels** (from `panels.json`) to show them.  

üìå *If no options are configured, users won't be able to create tickets!*  

### [üìú `config/panels.json`](/docs/reference/config/panels.mdx) ‚Äì Ticket Panels & Embeds \{#config-panels}
Panels are the **embeds/messages** that display the ticket options in Discord.  
- Each panel contains buttons/dropdowns linked to ticket options.  
- You can create **multiple panels** for different categories.  
- Panels can be fully customized with text, embeds, and layouts.  
- Panels can be linked to **specific options** (from `options.json`).

üìå *After configuring a panel, use the `/panel` command to send it to a channel!*  

### [üìÇ `config/transcripts.json`](/docs/reference/config/transcripts.mdx) ‚Äì Transcript Settings \{#config-transcripts}
This file controls **how ticket transcripts are saved** when a ticket is deleted.  
- Choose between **HTML or text transcripts**.  
- Customize the **transcript layout, colors, and format**.  
- Transcripts are disabled by default.

üìå *Transcripts let you keep a record of past tickets for future reference!*  

### üîß How to Edit the Config \{#config-edit}
1. Navigate to the `config/` folder inside your bot‚Äôs files.  
2. Open the JSON file you want to edit using a text editor.  
3. Modify the settings to fit your needs.  
4. Save your changes and restart the bot for them to take effect.  

üìå *Need more details? Check out the* [**full config reference**](/docs/category/api-config)!

**That‚Äôs it! You're ready to test the bot. üéâ**

## üí° Frequently Asked Questions \{#faq}
### How to add multiple questions, options or panels? \{#faq-multi-config}
To create multiple questions, options or panels, you need to <C color="red">**copy everything between and including the brackets (`{...}`)**</C>.  
Then you <C color="green">**paste it after the last question (`{...}`)**</C> and make sure that they are <C color="blue">**seperated by a comma**</C>.

### How to remove the website & reaction role options? \{#faq-remove-options}
To remove options that you don't want (e.g. website or reaction roles), you need to <C color="red">**delete everything between and including the brackets (`{...}`)**</C> from the option you want to remove.
Then you make sure that <C color="blue">**the comma also has been removed**</C>.

### Can I ignore or leave ... empty? \{#faq-empty-array}
Within the config, there are a lot of settings which are optional. These are most of the times lists/arrays (e.g. `["some list"]`), if you want to ignore it, you need to <C color="red">**leave it empty by setting it to (`[]`)**</C>.
Nothing else is required to be done, just make sure that <C color="green">**the list is empty**</C> by setting it to `[]`.

#### Use Cases
- **`general.json`** - `"globalAdmins"`, ...
- **`options.json`** - `"ticketAdmins"`, `"readonlyAdmins"`, `"questions"`, `"claimedCategory"`, ...
- **`panels.json`** - `"options"`, `"fields"`, ...
