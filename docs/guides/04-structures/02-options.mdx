## ⚙️ What Are Options? \{#about}
An **option** in Open Ticket is a customizable item that defines a ticket type, a website link, or a reaction role. You can compare it with the **button** of a panel.
Options allow you to create a flexible ticketing system where users can select from different options before submitting their tickets.

### 📌 Options Types \{#about-types}
There are currently 3 different option types in Open Ticket:
- **Ticket** – For creating a tickets.
- **Website** – For providing external links that users can access.
- **Role** – Reaction roles for allowing users to select roles via reactions.

### 🎯 Purpose of Options \{#about-purpose}
Options help with the following:
- ✅ Defining various ticket types, each with their own messages & customisation.
- ✅ Linking external websites for further information.
- ✅ Allowing users to select reaction roles for specific categories.

## 📍 Where Are Options Used? \{#usage}
Options are **defined in the `options.json` file** and referenced in the **`panels.json`** configuration. To use an option, its **`id`** must be added to the `"options"` array in `panels.json`.

- **File:** `options.json` → Defines available ticket options.  
- **File:** `panels.json` → Displays options in Discord as buttons or dropdown items.

## 🎨 Customization Options \{#customisation}
Options in Open Ticket can be **fully customized**. Here are some customization options for different types of options:

- **Tickets** – Choose the category, ticket admins, messages, embeds and additional settings for this ticket.
- **Websites** – Define a URL for a website that users can visit via a button.
- **Reaction Roles** – Set up roles that users can select by clicking a button.

### 💡 Best Practices \{#best-practises}
✔️ Use **clear and descriptive names** for ticket types.  
✔️ If not required, the website & reaction role options can be removed.  
✔️ You can create an unlimited amount of options. Each panel can only contain 25 options though.
