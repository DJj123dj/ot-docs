## ğŸ“‚ Database Structure \{#db-structure}
By default, Open Ticket stores all data inside the `./database/` folder, divided into <C color="blue">**five JSON files:**</C>

- **`global.json`** â€“ Stores global variables, including the **last Open Ticket version used**, IDs of auto-updating panels, and other general data.  
- **`tickets.json`** â€“ Stores all ticket-related data, including **creation time, participant IDs, ticket status and plugin data.**  
- **`users.json`** â€“ Currently <C color="red">**not used**</C> but reserved for plugins to store **custom user data.**  
- **`stats.json`** â€“ Holds **global, user, and ticket statistics** (e.g., total tickets created).  
- **`options.json`** â€“ Keeps a <C color="green">**backup of all ticket options**</C> used from the `options.json` config. If an option is missing from the config, it **retrieves it from here.**

## âœï¸ Manually Editing The Database \{#manual-edit}
Since Open Ticket stores data in **raw JSON files**, you can open and edit them using any **text editor** (VS Code, Notepad++, etc).

#### ğŸ“– JSON File Structure \{#json-structure}
All data in Open Ticket is structured as an **array of objects** using a `"category"`, `"key"`, and `"value"`:  
```json
[   {"category": "openticket:variable-type", "key": "user/message/ticket id", "value":null}   ]
```

- **`category`** â€“ Defines what type of variable is stored (e.g. `openticket:ticket`).  
- **`key`** â€“ The unique identifier, such as a **user ID, message ID, or ticket ID**.  
- **`value`** â€“ The stored data (could be a `string`, `number`, `boolean`, `object`, `array` or `null`).  

:::warning
- **Never modify the database structure** (keep the `category`, `key`, and `value` format).  
- **Back up your files** before making any manual changes.  
- **Stop the bot** before editing the database to prevent corruption.
:::

### â¬†ï¸ Updating Open Ticket \{#updating-openticket}
Follow these steps when you want to update Open Ticket:
1. **Copy the `./database/`, `./config/` & `./plugins/` folders** to another separate location.  
2. Update the bot by downloading a new version from Github.
3. Install all required npm dependencies and make sure the bot is able to start.
4. **Paste back the original `./database/`, `./config/` & `./plugins/` folders** in the new bot.
