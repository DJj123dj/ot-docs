## Overview
Using the `/autodelete` command, you can manage the autodelete system in a specific ticket.
You are able to disable it, (re-)enable it or change the inactivity time.
You can also provide an optional reason. This will be shown in the logs.

- **Default Permissions:** ðŸŸ  Ticket Admins
- **Usable Locations:** ðŸ”´ Ticket Only
- **Text Command:** `!ticket autodelete <disable|enable> <time> [reason]`
- **Slash Command:** `/autodelete <disable|enable> <time> [reason]`

## Parameters
- **`<disable|enable>`:** Enable or disable the autodelete system.
- **`<time>`:** Set the time the ticket needs to be inactive for when enabling the system. **<C color="green">(Only when using `enable`)</C>**
- **`[reason]`:** A reason for changing the autodelete system in the ticket. **<C color="green">(Optional)</C>**
