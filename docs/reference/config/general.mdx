# general.json <M color="cyan">object</M>
The `general.json` file serves as the main configuration for Open Ticket.
This file contains essential settings that define the bot's global behavior, appearance, and functionality across your server.

With `general.json`, you can fine-tune many aspects of your ticket system. Here are some examples:
- **Client configuration:** (token, status, server, ...)
- **Appearance:** (main color, visible buttons, enabled commands, ...)
- **Ticket System:** (features, permissions, extra's, ...)
- **Global Limits:** Set global limits for ticket creation.

If you need help with configuring the config, feel free to join our discord server!

## Info
These properties are only to give you some info. They don't serve a real purpose!
<Divider></Divider>
### `"_INFO"` <M color="cyan">object</M> \{#info}
**Default Value:** `/`

This object contains metadata for the config. This includes urls to our support server and the Open Ticket version this config is made for.

- **`"support"`:** <M color="blue">string</M> - A link to the Open Ticket documentation.
- **`"discord"`:** <M color="blue">string</M> - A link to the DJdj Development discord server.
- **`"version"`:** <M color="blue">string</M> - The Open Ticket version this config is made for.

:::warning version mismatch
If the version in the config is different from the version that you're using,
you might get a warning to re-check your config.
:::

**‚úÖ Join [our discord server](https://discord.dj-dj.be) when you need help setting up the bot!**

## Token
Configure your discord bot token here.
<Divider></Divider>
### `"token"` <M color="blue">string</M> \{#token}
**Default Value:** `"your bot token here"`<br></br>
**Allowed Values:** `A valid Discord token`

Insert your Discord bot token here. It is used as password to communicate with your Discord bot.

:::danger token safety
**NEVER EVER** share your Discord token! If someone is able to get your token, he/she can control the entire bot!<br></br>
<C color="red">**DJdj Development will never ask for your bot token!**</C>

#### Possible Concequences:
- Your server could get nuked/raided!
- Users can get roles they normally don't have access to!
- All messages can be logged! (also in staff channels)
- Users will be mass-banned!
- And much more...

#### Protecting your token:
- Never send a token to someone (DM and server)!
- Never upload a token to github. Put it inside a file which is covered by the `.gitignore` instead!
- Put the token inside a `.env` file!
- When you send the `general.json` to someone who wants to help you, make sure the token isn't included in the file!
:::

<Divider></Divider>
### `"tokenFromENV"` <M color="red">boolean</M> \{#token-env}
**Default Value:** `false`

Enable this boolean to read the token from a `.env` file instead of the `"token"` variable.
When you use this, you can leave the `"token"` variable empty!

```toml title=".env (Example)"
TOKEN=your-token-here
```

## Global Configuration
This is the main configuration of the bot. Think about the color of embeds, prefix and a few other things.
<Divider></Divider>
### `"mainColor"` <M color="blue">string</M> \{#main-color}
**Default Value:** `"#f8ba00"` (Open Ticket Yellow)<br></br>
**Allowed Values:** `A valid hex-color`

The main color of the bot. Used in almost all embeds of the bot.
Ticket messages and panels use their own custom color except when it isn't defined in the config.

**‚úÖ If you don't know how to get a hex-color, you can use the website [Color Picker](https://htmlcolorcodes.com/color-picker/) for it.**

<Divider></Divider>
### `"language"` <M color="blue">string</M> \{#language}
**Default Value:** `"english"`<br></br>
**Allowed Values:** `Check language list`

The language used in the bot. Almost 99% of all embeds, buttons and dropdowns in Open Ticket are translated.
**You can only select a language that is available in the list below.**

**‚úÖ If your language doesn't exist yet, feel free to contribute and add your language to the bot.**

<LinkBlock url="/docs/reference/languages" emoji="üá¨üáß" mode="emoji" name="Available Languages" description="Click here to view a list of all the available languages in Open Ticket!"></LinkBlock>

<Divider></Divider>
### `"prefix"` <M color="blue">string</M> \{#prefix}
**Default Value:** `"!ticket "` (With space at the end)<br></br>

The prefix for all the text commands in the bot (when enabled).
- **‚úÖ (Example 1)** If you have a prefix `!ticket `, the close command will be `!ticket close`
- **‚úÖ (Example 2)** If you have a prefix `!test`, the delete command will be `!testdelete`

<Divider></Divider>
### `"serverId"` <M color="blue">string</M> \{#server-id}
**Default Value:** `"discord server id"` (Invalid)<br></br>
**Allowed Values:** `Any valid discord server id`

In this variable, you need to put the id of the server that the bot will operate in.
It will be used for slash command creation and the database.

**‚úÖ Open Ticket is made for <C color="red">one server at a time</C>. It's impossible to use the bot in multiple servers at the same time.**

<Divider></Divider>
### `"globalAdmins"` <M color="pink">array</M> \{#global-admins}
**Default Value:** `["discord role id"]` (Invalid)<br></br>
**Allowed Values:** `List of valid discord role ids`

In this variable, you're able to put role id's from the server that have global admin permissions in the bot.
Global admins are able to control the ticket bot in every channel and use every command of the bot.

:::note Global Admin Permissions
- ‚úÖ Use all commands in all channels
- ‚úÖ View every ticket
- ‚úÖ Manage every ticket
- ‚úÖ Spawn panels
- ‚úÖ Blacklist users
- ‚úÖ Clear tickets
- ‚úÖ And more...
:::

**‚úÖ There are also <C color="red">Ticket Admins</C> available for roles that shouldn't have all these permissions.**

## Commands
Choose which type of commands you want to use in your server.
<Divider></Divider>
### `"slashCommands"` <M color="red">boolean</M> \{#slash-commands}
**Default Value:** `true` (Enabled)<br></br>

Enable slash commands in the bot. (Recommended)

**‚úÖ When you disable slash commands, they will all be removed from the bot.**

<Divider></Divider>
### `"textCommands"` <M color="red">boolean</M> \{#slash-commands}
**Default Value:** `true` (Enabled)<br></br>

Enable text commands in the bot. This allows you to use commands using the [**`"prefix"`**](#prefix).

**‚úÖ Make sure you've enabled the `MESSAGE_CONTENT` privileged gateaway intent in the discord dev panel.**

## Status
Set the status of the bot.
<Divider></Divider>
### `"status"` <M color="cyan">object</M> \{#status}

Set status of the bot. When enabled, you can configure and customise it to your preference. 

- **`"enabled"`:** <M color="red">boolean</M> - Enable a custom bot status.
- **`"type"`:** <M color="blue">string</M> - The status type. (`"listening"`, `"watching"`, `"playing"` or `"custom"`)
- **`"text"`:** <M color="blue">string</M> - The text that will be shown after the `"type"`. (recommended: `<30 characters`)
- **`"status"`:** <M color="blue">string</M> - The status of the bot. Check the list below for the available types.

> #### Status Options:
> - <C color="red">**üî¥ Do Not Disturb:**</C> `dnd`
> - <C color="orange">**üü† Idle:**</C> `idle`
> - <C color="green">**üü¢ Online:**</C> `online`
> - <C color="gray">**‚ö´Ô∏è Invisible:**</C> `invisible`

**‚úÖ Use the plugin [OT Better Status](https://github.com/DJj123dj/open-discord-plugins/tree/main/open-ticket/ot-better-status/) for a rotating status or variables inside the status!**

## Ticket System
Here you can configure everything related to the ticket system.
<Divider></Divider>
### `"system"."removeParticipantsOnClose"` <M color="red">boolean</M> \{#system-remove-participants-on-close}
**Default Value:** `false` (Disabled)<br></br>

When enabled, all participants (creator + extra participants) are removed from the ticket. They should be added back to the ticket when it is reopened.

**‚úÖ Global & ticket admins are not affected by this rule.**

<Divider></Divider>
### `"system"."replyOnTicketCreation"` <M color="red">boolean</M> \{#system-reply-on-ticket-creation}
**Default Value:** `true` (Enabled)<br></br>

When enabled, the bot will send an ephemeral message to the user that created a ticket.

<ScaleImage src="/img/docs-v4/creation-reply-message.png" size="40%" alt="The reply you receive when you create a ticket."></ScaleImage>

**‚úÖ When using the `/ticket` command, this message will always be created. It is not affected by this rule.**

<Divider></Divider>
### `"system"."replyOnReactionRole"` <M color="red">boolean</M> \{#system-reply-on-reaction-role}
**Default Value:** `true` (Enabled)<br></br>

When enabled, the bot will send an ephemeral message to the user when he/she updates here roles via the built-in reaction roles.

<ScaleImage src="/img/docs-v4/reaction-role-add.png" size="40%" alt="The reply you receive when you update your roles."></ScaleImage>

**‚úÖ The bot will show which roles have been added and which ones have been removed.**

<Divider></Divider>
### `"system"."useTranslatedConfigChecker"` <M color="red">boolean</M> \{#system-use-translated-config-checker}
**Default Value:** `true` (Enabled)<br></br>

Enable this variable to translate the config checker in the console. This could be helpful if you don't understand the error which is caused by the config.

<ScaleImage src="/img/docs-v4/config-checker.png" size="60%" alt="An example of a config checker report in the console or cmd."></ScaleImage>

**‚úÖ This report in the console will be translated.**

<Divider></Divider>
### `"system"."preferSlashOverText"` <M color="red">boolean</M> \{#system-prefer-slash-over-text}
**Default Value:** `true` (Enabled)<br></br>

When enabled, the bot will prefer slash commands over traditional text commands in the help menu and related commands.

**‚úÖ This won't disable slash or text commands. It's only made for displaying purposes.**

<Divider></Divider>
### `"system"."sendErrorOnUnknownCommand"` <M color="red">boolean</M> \{#system-send-error-on-unknown-command}
**Default Value:** `true` (Enabled)<br></br>

If enabled, the bot will respond with an error message when users enter the prefix without an unknown command.
This can help guide users to the correct commands.

<ScaleImage src="/img/docs-v4/unknown-command.png" size="40%" alt="Someone tried an unknown command."></ScaleImage>

**‚úÖ Users will be notified of invalid commands.**

<Divider></Divider>
### `"system"."questionFieldsInCodeBlock"` <M color="red">boolean</M> \{#system-question-fields-in-code-block}
**Default Value:** `true` (Enabled)<br></br>

When enabled, the bot will format question fields in code blocks for better readability in the ticket message.

<ScaleImage src="/img/docs-v4/ticket-message-questions.png" size="40%" alt="A code block question field in the ticket message."></ScaleImage>

**‚úÖ Improves the clarity of question response fields.**

<Divider></Divider>
### `"system"."disableVerifyBars"` <M color="red">boolean</M> \{#system-disable-verify-bars}
**Default Value:** `false` (Disabled)<br></br>

Enable this to disable the verify bars in the ticket system. Verify bars are the (‚ùå,‚úÖ) buttons you see when clicking on an action.

<ScaleImage src="/img/docs-v4/verifybar.png" size="40%" alt="An example of a verifybar. Some bars might look different based on the action."></ScaleImage>

**‚úÖ When disabling this, you're also unable to choose a reason when using the button.**

<Divider></Divider>
### `"system"."useRedErrorEmbeds"` <M color="red">boolean</M> \{#system-use-red-error-embeds}
**Default Value:** `true` (Enabled)<br></br>

Enable this to display error message embeds with a red color instead of the default color, making errors more noticeable to users.

**‚úÖ Red embeds improve the visibility of error messages.**

<Divider></Divider>
### `"system"."emojiStyle"` <M color="blue">string</M> \{#system-emoji-style}
**Default Value:** `"before"`<br></br>
**Allowed Values:** `"before"`, `"after"`, `"double"` or `"disabled"`

This variable determines where emojis appear in message titles, embed titles, subtitles & more.

|Value               |Embed Title                     |
|--------------------|--------------------------------|
|`before` (default)  |**‚úÖ  Example Embed Title**     |
|`after`             |**Example Embed Title ‚úÖ**      |
|`double`            |**‚úÖ Example Embed Title ‚úÖ**    |
|`disabled`          |**Example Embed Title**         |

**‚úÖ Customize emoji placement to suit your server‚Äôs style.**

<Divider></Divider>
### `"system"."enableTicketClaimButtons"` <M color="red">boolean</M> \{#system-enable-ticket-claim-buttons}
**Default Value:** `true` (Enabled)<br></br>

When enabled, claim & unclaim buttons will appear on tickets, allowing team members to claim responsibility for a ticket.

**‚úÖ This does not affect the `/claim` & `/unclaim` commands!**

<Divider></Divider>

### `"system"."enableTicketCloseButtons"` <M color="red">boolean</M> \{#system-enable-ticket-close-buttons}
**Default Value:** `true` (Enabled)<br></br>

When enabled, close & reopen buttons will appear on tickets, allowing quick closure without commands.

**‚úÖ This does not affect the `/close` & `/reopen` commands!**

<Divider></Divider>
### `"system"."enableTicketPinButtons"` <M color="red">boolean</M> \{#system-enable-ticket-pin-buttons}
**Default Value:** `true` (Enabled)<br></br>

When enabled, pin & unpin buttons will appear on tickets, allowing team members to pin important messages in a ticket channel.

**‚úÖ This does not affect the `/pin` & `/unpin` commands!**

<Divider></Divider>
### `"system"."enableTicketDeleteButtons"` <M color="red">boolean</M> \{#system-enable-ticket-delete-buttons}
**Default Value:** `true` (Enabled)<br></br>

When enabled, a delete button will appear on tickets, delete tickets quickly.

**‚úÖ This does not affect the `/delete` command!**

<Divider></Divider>
### `"system"."enableTicketActionWithReason"` <M color="red">boolean</M> \{#system-enable-ticket-action-with-reason}
**Default Value:** `true` (Enabled)<br></br>

When enabled, you're able to give a reason for a specific ticket action.

:::note supported actions
These are the supported actions which are compatible with reasons:
- Claim & Unclaim
- Pin & Unpin
- Close & Reopen
- Delete
- Add, Remove & Blacklist (User)
- Move & Rename
- Autoclose & Autodelete
:::

**‚úÖ The given reasons will also show up in the logs.**

<Divider></Divider>
### `"system"."enableDeleteWithoutTranscript"` <M color="red">boolean</M> \{#system-enable-delete-without-transcript}
**Default Value:** `true` (Enabled)<br></br>

Enable this to allow tickets to be deleted without generating a transcript. Use with caution, as it removes any record of the ticket.

**‚úÖ Ideal for tickets with sensitive information that shouldn't be recorded.**

<Divider></Divider>

Log almost all bot events and actions to a specific discord channel.<br></br>

- **`"enabled"`:** <M color="red">boolean</M> - Enable logging.
- **`"channel"`:** <M color="blue">string</M> - The Discord channel where logs will be sent. (valid channel ID) 

<ScaleImage src="/img/docs-v4/logs-discord.png" size="50%" alt="An example of a claim log message in a discord channel."></ScaleImage>

## Global Ticket Limits
Global ticket limits are limits you can configure which will apply to each ticket option.
<Divider></Divider>
### `"system"."limits"` <M color="cyan">object</M> \{#system-limits}

Enable this to restrict the number of tickets that can be opened simultaneously. This is great to prevent overloading the ticket system.
Individual limits per option can also be configured in the **[`options.json`](./options.mdx#ticket-limits)** config.

- **`"enabled"`:** <M color="red">boolean</M> - Enable global ticket limits.
- **`"globalMaximum"`:** <M color="green">number</M> - The maximum number of tickets that can be open globally across the bot.
- **`"userMaximum"`:** <M color="green">number</M> - The maximum number of tickets a single user can have open at any given time.

**‚úÖ This will apply for all existing tickets, including `closed` ones.**

## Permissions
Configure permissions for individual commands.
<Divider></Divider>
### `"system"."permissions"` <M color="cyan">object</M> \{#system-permissions}
**Default Value:** `/`<br></br>
**Allowed Values:** `"everyone"`, `"admin"`, `"none"` or `Any valid discord role id`

This property determines the access levels for various bot commands and actions.
Use this to customize permissions for different roles or IDs.

**Available options are:**
- `"everyone"`: Allow for everyone.
- `"admin"`: Allow for `global admins` and `ticket admins`.
- `"<role id>"`: Allow for a specific discord role ID.
- `"none"`: Don't allow usage for everyone. <C color="red">**This will also remove the slash command & help menu command.**</C>

| **Command/Action** | **Default Value** | **Description** |
|---------------------|-------------------|-----------------|
| `help`             | `everyone`       | Access to the help command. |
| `panel`            | `admin`          | Access to the control panel. |
| `ticket`           | `none`           | Permission to create tickets. |
| `close`            | `everyone`       | Permission to close tickets. |
| `delete`           | `admin`          | Permission to delete tickets. |
| `reopen`           | `everyone`       | Permission to reopen tickets. |
| `claim`            | `admin`          | Permission to claim tickets. |
| `unclaim`          | `admin`          | Permission to unclaim tickets. |
| `pin`              | `admin`          | Permission to pin messages in tickets. |
| `unpin`            | `admin`          | Permission to unpin messages in tickets. |
| `move`             | `admin`          | Permission to move tickets to a different category. |
| `rename`           | `admin`          | Permission to rename tickets. |
| `add`              | `admin`          | Permission to add users to tickets. |
| `remove`           | `admin`          | Permission to remove users from tickets. |
| `blacklist`        | `admin`          | Permission to blacklist users from opening tickets. |
| `stats`            | `everyone`       | Access to view stats. |
| `clear`            | `admin`          | Permission to purge multiple tickets. |
| `autoclose`        | `admin`          | Permission to configure auto-closing of tickets. |
| `autodelete`       | `admin`          | Permission to configure auto-deletion of tickets. |

**‚úÖ Customize each permission to suit your server's requirements.**

## Messages
Configure logs and DM messages for individual actions.
<Divider></Divider>
### `"system"."messages"` <M color="cyan">object</M> \{#system-messages}
**Default Value:** `{"dm":false, "logs":true}`<br></br>

This property controls whether specific actions and commands are logged in the server and/or sent as a DM to the ticket creator.
The configuration allows you to enable or disable notifications for each action.

| **Action/Command**    | **Default (DM)** | **Default (Logs)** | **Description**                               |
|-----------------------|------------------|--------------------|-----------------------------------------------|
| `creation`            | `false`          | `true`             | Notification when a ticket is created.       |
| `closing`             | `false`          | `true`             | Notification when a ticket is closed.        |
| `deleting`            | `false`          | `true`             | Notification when a ticket is deleted.       |
| `reopening`           | `false`          | `true`             | Notification when a ticket is reopened.      |
| `claiming`            | `false`          | `true`             | Notification when a ticket is claimed.       |
| `pinning`             | `false`          | `true`             | Notification when a ticket is pinned.        |
| `adding`              | `false`          | `true`             | Notification when a user is added to a ticket.|
| `removing`            | `false`          | `true`             | Notification when a user is removed from a ticket.|
| `renaming`            | `false`          | `true`             | Notification when a ticket is renamed.       |
| `moving`              | `false`          | `true`             | Notification when a ticket is moved.         |
| `blacklisting`        | `false`          | `true`             | Notification when a user is blacklisted.     |
| `roleAdding`          | `false`          | `true`             | Notification when a role is given to a user. |
| `roleRemoving`        | `false`          | `true`             | Notification when a role is removed from a user.|

**‚úÖ Customize these options to control how and where users are notified about ticket actions.**
